Fluxlab

Complete

A VR circuit building game built in the Unreal Engine

Latest Release

Origin

Fluxlab was my submission for the final project of CS4331: Virtual Reality.

Overview

Fluxlab is a VR circuit building game built in the Unreal Engine. The game allows players to build complex circuits along a grid using 4 different blocks.

Systems

Every aspect of the game was written in C++ by me. This includes, the grid system, power system, arm menu, and blocks.

Superfist

Complete

A silly 3D fighting game built in the Unity Engine

Latest Release

Origin

Superfist is a game I made during HackwestTX 2020.

Overview

Superfist is a 3D fighting game built in the Unity Engine about ordinary people with extraordinary punching ability.

Systems

The fighting system was bought from the unity asset store and heavily modified by me to support Unity's new input system, block sliding, and ragdolling.

Tamagotchi

Complete

A virtual pet game built in Javatech, a custom Java game engine.

Latest Release

Origin

Tamagotchi was my submission for the final project of CS4365: Software Engineering II.

Overview

Tamagotchi allows you to take care of and communicate with a virtual pet. Your pet must eat, drink, be cleaned, and be played with in order to stay happy and alive.

Systems

The game was built upon JavaTech, a rudimentary custom game engine written by myself.

Specular

Complete

A system that allows writers to write complex dialogue scenes to be run in game engines

Latest Release

SpecularLang (Slang)

Slang is a custom programming language built using Antlr4. It allows for conditionals, choices, dialogue, custom actions, and math.

Implementations

Specular has been implemented into the Unreal Engine as a plugin. The plugin is currently closed source.

Project Requirements

The Project is based on the 2021 Student Game Competition. Your game must follow one of the following categories.

  1. Innovative Interfaces: Games submitted to this category should be games that push the boundaries of current games and game interface practice. Example areas include the use of gesture, multi-touch, multi-screen or haptics; voice input; use of sensors such as breathing or heart rate; augmented reality games, mixed reality games, novel use of game AI and machine learning techniques, or explore novel, thought-provoking forms of interaction.
  2. Transformative & Transgressive Play: This category attends to the expanding boundaries of play and game design with the emergence of independent games, art games, and political games as well as gamified/playbourised systems that explore the role of play in non-gaming settings. For example, games now play an increasingly important role in areas like education, healthcare, safety, urban planning, sustainability, and other economic, cultural, and societal sectors. Games submitted to this category should draw on these expanding boundaries of game design to create innovative gameplay for non-gaming settings, and critically reflect on challenges.
